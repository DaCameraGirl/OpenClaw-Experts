# Setup Checklist

Use this before starting real tasking.

## Step 1: Qualify

- [ ] Read all onboarding instructions.
- [ ] Take and pass the assessment.
- [ ] Open Project Obsidian from the HAI platform.
- [ ] Copy the assigned `@obsidian.handshakecommunity.ai` credential email from the Project Obsidian page.

## Step 2: Multimango Setup

- [ ] Go to `multimango.com`.
- [ ] Create/sign into Multimango with the assigned `@obsidian.handshakecommunity.ai` credential email.
- [ ] Use the confirmation code forwarded to the personal email.
- [ ] Do not use a personal email for Multimango task submission.
- [ ] If signed into Multimango from another project, sign out and sign back in with the Project Obsidian credential.
- [ ] Wait to be added to the Claw Onboarding group if no tasks appear.
- [ ] Expect the Claw Onboarding group wait to take 24-48 hours.
- [ ] Once added, confirm `Claw Onboarding` and `Claw Annotate` are visible.

Important: work will not flow back to HAI and will not be paid unless Multimango uses the assigned `@obsidian.handshakecommunity.ai` credential.

## Step 3: OpenClaw Onboarding

- [ ] Select the `Claw Onboarding` task in Multimango.
- [ ] Do not claim a HAI task or start the HAI timer for the Multimango onboarding step.
- [ ] Set up the OpenClaw environment.
- [ ] Connect Telegram for mobile OpenClaw use.
- [ ] Bookmark the OpenClaw VM URL.
- [ ] Open OpenClaw from the VM link.
- [ ] Complete `USER.md`.
- [ ] Complete `IDENTITY.md`.
- [ ] Complete `SOUL.md`.
- [ ] Spend at least 30 minutes on the memory files.
- [ ] Use personal email for real service connections, not the `@obsidian.handshakecommunity.ai` credential.
- [ ] Connect 2-3 standard OpenClaw services Angela is comfortable using.
- [ ] Prefer Google Workspace first for calendar, read-only email if available, Drive, Docs, and Sheets.
- [ ] Connect at least one low-risk additional tool if available:
  - Whoop
  - NewsAPI / News Summary
  - Wikimedia Commons
  - Wikipedia
  - Finnhub
  - Hacker News
  - Typeform
  - Mailchimp
  - FTC ReportFraud
- [ ] Confirm onboarding is complete in the OpenClaw dashboard.

## Tool Connection Rules

- [ ] Google Workspace is connected.
- [ ] Telegram is connected.
- [ ] 2-3 standard services are connected if onboarding requires them.
- [ ] Tools are connected directly through OpenClaw.
- [ ] Composio or third-party connector services are not used.
- [ ] WhatsApp and other social media apps are not connected.

## Tool Verification

For each connected tool:

- [ ] Confirm OpenClaw can see the tool.
- [ ] Run a low-risk read-only check when possible.
- [ ] For state-changing tools, use harmless test data first.
- [ ] Disconnect tools that are no longer actively needed.

## Per-Task Startup

- [ ] Claim one HAI task.
- [ ] Start the HAI timer before opening Multimango or the VM.
- [ ] Pick one Multimango recommended tool/topic row.
- [ ] Write the prompt in your own words.
- [ ] Include a concrete state change when possible.
- [ ] Plan at least one follow-up interaction with OpenClaw.

## Daily Expectations

- [ ] Submit at least 2 rubrics per day to stay active.
- [ ] Read the Multimango Guidance panel at every step.
- [ ] Follow all in-flow compliance rules.
- [ ] Try at least one difficult prompt per day when possible.
- [ ] Before a tricky prompt, tag the message with the correct OpenClaw category.
- [ ] Keep follow-ups on the same tag/topic.
- [ ] Clear the tag if switching topics.
- [ ] After the task is complete, mark it complete and annotate it.
- [ ] Include the Safety & Trustworthiness category when annotating from the OpenClaw VM flow.

## Connector Health

- [ ] Keep the Fix Connectors list empty.
- [ ] Click Fix on broken connectors and resolve them.
- [ ] Search the pre-configured New Tools list before trying another setup route.
- [ ] If a tool is not listed, ask OpenClaw: `Please help me setup <toolname>`.

## Feedback And Bugs

- [ ] Review Multimango submission feedback regularly.
- [ ] Acknowledge feedback on tasks when required.
- [ ] Check `troubleshooting-notes.md` if grading stays at `0%`, feedback does not appear, or onboarding access is delayed.
- [ ] For 503 errors, spinning VM, broken verifiers, or annotation failures, use the MM Downtime or Bug Report Slack workflow.

## Bonus Note

- [ ] After onboarding and one completed real task, the project says there is a $22 onboarding bonus.
